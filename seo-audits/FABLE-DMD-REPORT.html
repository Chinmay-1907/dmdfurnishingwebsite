What we didThe 9 iterations, commit by commit

Iteration 1 — fix the emergencies 3c3610b

Logo was a 2 MB file loading on every page — now 16 KB. Fixed 133 image links that worked on Windows but would 404 in production (wrong upper/lower case). Built a checker script so this class of bug can never ship again.

Iteration 2 — visibility & conversion 60e8ad6

~150 of 174 product pages were invisible to crawlers (pagination hid them) — added A–Z crawl indexes. Made the contact page form-first. Added llms.txt + RSS + speakable markup so AI search engines (ChatGPT, Perplexity) can read the site. Wrote per-group product copy across all 174 product pages. Root-caused the hotel page's crashed performance audit (an auto-scrolling menu on page load).

Iteration 3 — structure d96dac7

Built 20 new mid-tier landing pages (/products/hotel/guest-room and the like) wiring place → furniture-type → product. Contact page made fully static (SEO 92 → 100). First accessibility contrast batch.

Iteration 4 — motion & URLs e733981

Removed infinite background animations that kept the page "visually busy" forever (projects page speed index 10.1 s → 1.5 s). Inspiration pages moved from /inspirations/3 to readable URLs like /inspirations/modern-minimalist, with 301 redirects. Schema honesty pass: real image sizes, removed fake weekend hours.

Iteration 5 — accessibility 100 a83064c

Found why 4 pages still failed accessibility: the hidden mobile menu was technically still "on the page", and keyboard users could Tab into invisible links. Fixed properly. Switched font loading to eliminate the text-reflow flash. Result: SEO / Accessibility / Best Practices all 100, all 13 pages.

Iteration 6 — image bugs & paint speed 5e2b4d0

First hero slide now paints instantly instead of fading from black. First three catalog cards load eagerly (they're the page's main content). Found and fixed 9 images whose filenames contained a special dash character that silently broke them — renamed, references rewritten, checker upgraded to catch it forever.

Iteration 7 — design team pass f317284 + fb5bb63

Full design review (details below): five UI improvements shipped, zero critical findings on the 41-rule design linter, re-audit confirmed no score regression.

Iteration 8 — blog images ef0148a

All six blog posts were walls of text. Each now has two photo figures with captions — and every photo was opened and inspected before its description was written (three filenames were lying about their contents).

Iteration 8b — schema images 640ff19

Each post's structured data now lists its actual images instead of one shared generic photo, so search engines can cite them.

Design team reviewUI/UX pass — every agent lens, every rule

Reviewed against the design team's full rulebook: the 7 team rules, the 27-rule anti-slop checklist, the 41-rule impeccable linter, and the critic's visual scorecard — across 6 key pages on desktop (1440×900) and mobile (375×812), plus live checks of every interactive element.

Shipped

  • SHIPPEDHeadline copy: "Custom Hospitality Furniture. Designed. Manufactured. Delivered." — dropped the em-dash (banned character), kept the gold drop-caps.
  • SHIPPED"Get Started" labels replaced with specific copy: "Plan Your Project" (services) and "Work With Us" (projects) — generic CTAs are on the forbidden list.
  • SHIPPEDProduct pages: photo follows you. The image card used to end high and leave a dead white hole while specs kept scrolling. It now sticks below the header and rides alongside.

  • SHIPPEDReduced-motion respect: the "285+ rooms" counters no longer animate for users whose system requests less motion.

  • KEPTBefore/after room slider, count-up stats, contact map — all three looked broken in automated captures and all three were verified working live (lazy-loading caught mid-screenshot).
